## Version 1.2.1

This release marks a significant step forward for al-muslim. It addresses every bug and quality issue reported since the initial release, introduces meaningful UI improvements, and lays the groundwork for a major architectural change and multi-platform expansion.

---

## Bug Fixes

Every issue reported by the community has been reviewed, reproduced, and addressed in this release.

### Critical

**#11 — Encoding Errors in Quran Text**
Several Surahs were displaying black diamond replacement characters (�) between words due to the font lacking full coverage of the Quranic Unicode range. The fix involved switching to a font with complete Quranic support and ensuring all Quran text is consistently encoded as UTF-8 throughout the codebase. This was treated as the highest priority fix in this release.

### UI & Layout Bugs

**#1 — App Icon Misaligned**
The app icon had a visible white void on the bottom-left edge, with the logo shifted toward the top-right. This was caused by incorrect adaptive icon bounds and padding. The icon asset and its configuration have been corrected.

**#2 — Adhkar Category Bar Misalignment & Text Truncation**
The category selection bar in the Adhkar page was pushed to the left, leaving empty space on the right, and tab labels were being clipped (e.g. "Morning" appearing as "Morn..."). The tab layout now distributes evenly across the full screen width and labels are no longer truncated.

**#3 — Theme State Not Applying Beyond Settings**
The Dark/Light theme toggle was only updating the Settings screen. Navigating to any other page (Home, Prayer, Adhkar, etc.) left the app in its default Light state. The theme state is now reactive app-wide and persists correctly across all screens.

**#4 — Misaligned Button in Settings Header Account Section**
A button in the Account section of the Settings header was being clipped by the left edge of the screen, making it impossible to interact with. The layout constraints have been corrected.

**#8 — Lack of Landscape & Tablet Optimization**
When running on large screens or in landscape orientation, the UI did not adapt and showed large black bars on both sides. As a temporary measure, the app now explicitly locks to portrait mode to prevent the broken state. Full responsive layout support for tablets and landscape is planned for a future release.

## What's Changing in This Release

### UI Overhaul

The interface is being reworked to feel more intentional and consistent. This includes fixing existing layout inconsistencies, improving spacing and visual hierarchy, and ensuring the app looks and behaves correctly across different screen sizes and Android versions. The goal is a cleaner, more polished experience without changing what makes the app familiar.

### Architecture: Exploring a Migration from React Native to Kotlin

The Android version of al-muslim was originally built with React Native. While this allowed for rapid development, it comes with real trade-offs in performance, startup time, and how naturally the app integrates with Android's UI system.

We are actively evaluating a full migration to **Kotlin with Jetpack Compose**. This would mean:

- Faster startup and smoother animations
- Better memory efficiency and battery usage
- A UI that behaves exactly like a native Android app
- Easier access to Android-specific APIs (wake lock, haptics, notifications, etc.)

This migration will not happen overnight. The plan is to rebuild screen by screen, ensuring nothing regresses in the process. More details will be shared as the work progresses.

## Upcoming Features

The following feature requests were submitted by the community and are confirmed for upcoming releases:

**#5 — Haptic Feedback on Adhkar Counter**
A vibration pulse will be added on each counter tap, with a distinct pattern when the target count is reached. This will be controllable from Settings, including intensity adjustment and the option to disable it entirely.

**#6 — AMOLED Dark Mode**
A true black (#000000) background option will be added as a separate theme alongside the existing dark mode. This is particularly beneficial for OLED screens — individual pixels turn off for pure black, which reduces power consumption and lowers the risk of burn-in.

**#7 — 12-Hour AM/PM Time Format**
Prayer times are currently displayed in 24-hour format across the app and in notifications. A toggle will be added to Settings to switch to 12-hour format, applying consistently everywhere times appear.

**#9 — Copy Feature for Surahs & Adhkar**
Users will be able to copy any Surah verse or Adhkar text to the clipboard directly from the reading view, either via a long-press or a dedicated copy button.

**#10 — Prevent Screen Sleep on Quran & Adhkar Pages**
The screen will stay awake while the Quran or Adhkar pages are open, so users don't have to keep touching the display while reading or reciting. A toggle will be available in Settings for users who prefer the default sleep behavior.

## Platform Availability

al-muslim is expanding beyond Android. Here is the current and planned platform status:

### Android
Available now. The primary platform. Download the latest APK from the [Releases](https://github.com/lnfiniware/al-muslim/releases) page.

### Linux
Available now. The Linux version is a separate, dedicated application built with **QML**, designed to feel native on the Linux desktop.

Repository: [github.com/lnfiniware/al-muslim-for-Linux](https://github.com/lnfiniware/al-muslim-for-Linux)

### iOS & iPadOS
In development. The iOS and iPadOS version is being built to take full advantage of Apple's platform conventions. iPadOS support will include an optimized layout for larger screens from the start.

### Windows
In development. A native Windows application is planned, with a UI appropriate for desktop use rather than a simple port of the mobile layout.

### macOS
In development. macOS support is planned alongside Windows, with attention to how the app fits within the Mac desktop environment.
